Design a module called TopModule. This module is an edge detector that synchronizes an input signal and emits pulse outputs for rising and falling edges.

## Overview

TopModule detects transitions (rising and falling edges) on a width-bit input signal. It synchronizes the input to the local clock domain using a two-stage flip-flop chain (if EnSync is enabled), then compares consecutive synchronized values to detect edges. For each detected edge, a one-cycle pulse is emitted on the appropriate output.

## Parameters

| Parameter | Meaning | Default |
|-----------|---------|---------|
| `Width` | Width of the input signal, in bits. | 1 |
| `ResetValue` | Initial state of synchronized signal on reset. | 0 |
| `EnSync` | When 1, enable two-stage synchronizer; when 0, no synchronization. | 1 |

## Interface

| Port | Direction | Width | Description |
|------|-----------|-------|-------------|
| `clk_i` | input | 1 | System clock (used for edge detection and synchronization). |
| `rst_ni` | input | 1 | Asynchronous active-low reset. |
| `d_i` | input | Width | Input signal to monitor for edges. |
| `q_sync_o` | output | Width | Synchronized input value (output of the synchronizer). |
| `q_posedge_pulse_o` | output | Width | Rising-edge pulse: bit j is high for one cycle when q_sync_o[j] transitions from 0 to 1. |
| `q_negedge_pulse_o` | output | Width | Falling-edge pulse: bit j is high for one cycle when q_sync_o[j] transitions from 1 to 0. |

## Behavioral requirements

### Synchronization Stage
- When `EnSync = 1`, a two-stage flip-flop synchronizer is instantiated:
  - Stage 1 captures `d_i` on the clock edge.
  - Stage 2 captures the output of stage 1 on the next clock edge.
  - `q_sync_o` reflects the output of stage 2 (synchronized with 2-cycle latency).
- When `EnSync = 0`, no synchronizer is used:
  - `q_sync_o` is directly assigned to `d_i` (combinational).

### Edge Detection
- A third flip-flop captures `q_sync_o` on each clock edge, creating a delayed version.
- Rising-edge pulses are generated by: `q_posedge_pulse_o = q_sync_o & ~q_prev` (current high AND previous low).
- Falling-edge pulses are generated by: `q_negedge_pulse_o = ~q_sync_o & q_prev` (current low AND previous high).
- Pulses are asserted for exactly one cycle when a transition is detected.

### Per-Bit Detection
- Edge detection is per-bit: each bit of the input is independently monitored.
- Transitions on different bits produce independent pulses on the output vectors.

### Reset Behavior
- On `rst_ni` assertion (active low):
  - The synchronizer pipeline (if enabled) is reset to `ResetValue`.
  - The delayed register (for edge detection) is reset to `ResetValue`.
  - No pulses are emitted during the first cycle after reset (since both current and previous values are the same).

### Latency
- When `EnSync = 1`: Detection latency is 3 cycles (two synchronizer stages plus one compare stage).
  - Cycle 0: `d_i` changes.
  - Cycle 1: Stage 1 of synchronizer captures the change.
  - Cycle 2: Stage 2 (q_sync_o) reflects the change.
  - Cycle 3: Edge pulse is emitted (current != previous).
- When `EnSync = 0`: Detection latency is 1 cycle.
  - Cycle 0: `d_i` changes.
  - Cycle 1: Edge pulse is emitted.

### Multiple Edges
- If the input transitions multiple times in consecutive cycles, each transition generates a corresponding pulse.
- Pulses never overlap for a single bit (a bit cannot be high and low in the same cycle).

## Example

With `Width = 1`, `EnSync = 1`, `ResetValue = 0`:

| Cycle | `d_i` | q_sync_o (after stage 2) | Prev Register | q_posedge_pulse_o | q_negedge_pulse_o | Notes |
|-------|---|---|---|---|---|---|
| 0 | 0 | 0 | 0 | 0 | 0 | Reset state |
| 1 | 1 | 0 | 0 | 0 | 0 | Input changes to 1; stage 1 captures |
| 2 | 1 | 0 | 0 | 0 | 0 | Stage 2 gets stage 1 value (still 0) |
| 3 | 1 | 1 | 0 | 1 | 0 | q_sync_o becomes 1; edge detected (1 & ~0) |
| 4 | 1 | 1 | 1 | 0 | 0 | No change; pulse deasserts |
| 5 | 0 | 1 | 1 | 0 | 0 | Input changes to 0; stage 1 captures |
| 6 | 0 | 1 | 1 | 0 | 0 | Stage 2 still high |
| 7 | 0 | 0 | 1 | 0 | 1 | q_sync_o becomes 0; falling edge detected (~0 & 1) |
| 8 | 0 | 0 | 0 | 0 | 0 | No change; pulse deasserts |

With `EnSync = 0` (no synchronizer), detection happens in cycle 1 (one cycle latency).
