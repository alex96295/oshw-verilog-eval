Design a module called TopModule. This module is a programmable pattern generator: it
drives two independent channels (each with clock and data outputs) with patterns configured
via memory-mapped registers, and reports completion via interrupts.

Overview

TopModule is a register-controlled pattern generator that drives configurable clock and data
signals on two independent output channels. Each channel can be programmed with different
timing and data patterns via a TileLink memory-mapped register interface. The module operates
synchronously with respect to an input clock; it generates its own output clocks and data
sequences based on register settings. Completion of a pattern is signaled via interrupt pins.
The module also supports alert reporting via dedicated alert interface pairs.

Parameters

  Parameter   Meaning   Constraint  
 ----------- --------- ------------ 
  AlertAsyncOn   Mask vector enabling asynchronous alert reporting for each alert source.   Width is NumAlerts; each bit selects async (1) or sync (0) alert response.  
  AlertSkewCycles   Clock cycles of skew introduced in alert timing.   Typically 1; adjusts alert responsiveness.  

Interface

  Port   Direction   Width   Description  
 ------ ----------- ------- ------------- 
  clk_i   input   1   System clock for register interface and pattern timing.  
  rst_ni   input   1   Active-low asynchronous reset.  
  tl_i   input   tlul_h2d_t   TileLink host-to-device; carries register write/read commands from the host.  
  tl_o   output   tlul_d2h_t   TileLink device-to-host; carries responses to register accesses.  
  alert_rx_i   input   alert_rx_t[NumAlerts-1:0]   Alert RX array; incoming ping/ack handshake signals for alert response protocol.  
  alert_tx_o   output   alert_tx_t[NumAlerts-1:0]   Alert TX array; outgoing alert assertion and acknowledgment signals.  
  cio_pda0_tx_o   output   1   Pattern data A channel 0 output; drives data pattern on channel 0.  
  cio_pcl0_tx_o   output   1   Pattern clock channel 0 output; drives clock for channel 0.  
  cio_pda1_tx_o   output   1   Pattern data A channel 1 output; drives data pattern on channel 1.  
  cio_pcl1_tx_o   output   1   Pattern clock channel 1 output; drives clock for channel 1.  
  cio_pda0_tx_en_o   output   1   Output enable for cio_pda0_tx_o; high to drive the pin.  
  cio_pcl0_tx_en_o   output   1   Output enable for cio_pcl0_tx_o; high to drive the pin.  
  cio_pda1_tx_en_o   output   1   Output enable for cio_pda1_tx_o; high to drive the pin.  
  cio_pcl1_tx_en_o   output   1   Output enable for cio_pcl1_tx_o; high to drive the pin.  
  intr_done_ch0_o   output   1   Interrupt signal for channel 0; asserted when pattern generation on channel 0 completes.  
  intr_done_ch1_o   output   1   Interrupt signal for channel 1; asserted when pattern generation on channel 1 completes.  

Behavioral requirements

- Register interface. TopModule provides a set of memory-mapped registers accessible via TileLink. These registers control the pattern on each channel: clock period, data sequence, repetition count, and enable flags. Register reads and writes follow standard TileLink protocol with ready/valid handshaking.

- Two independent channels. Each channel (0 and 1) operates independently with its own configuration, clock generator, and data sequence. Patterns can run concurrently or sequentially based on register settings.

- Clock generation. Each channel clock (cio_pcl0_tx_o and cio_pcl1_tx_o) is generated by dividing the system clock according to the channel's clock-divider register. The output is a square wave with period controlled by register settings.

- Data pattern generation. Each channel data output (cio_pda0_tx_o and cio_pda1_tx_o) drives a programmable bit sequence, typically with synchronization to the generated clock. The data pattern is loaded from registers and may be repeated.

- Pattern completion and interrupts. When a pattern finishes (reaches the programmed repetition count or when explicitly stopped), the corresponding interrupt (intr_done_ch0_o or intr_done_ch1_o) is asserted. Interrupts persist until cleared via register write.

- Output enable control. Each pin has a separate output-enable signal (cio_pda0_tx_en_o, cio_pcl0_tx_en_o, cio_pda1_tx_en_o, cio_pcl1_tx_en_o). These allow tri-state control or open-drain driving if needed, independent of the data/clock values.

- Alert reporting. The module supports multiple alert sources (e.g., configuration errors, timing violations). Alerts are signaled via the alert_tx_o interface with ping/ack handshaking. Behavior (synchronous vs. asynchronous) is controlled by the AlertAsyncOn parameter.

Example

Configure channel 0 for a square-wave clock with 50% duty cycle and 16-bit data pattern:
- Write registers: clk_div[0] = 2 (clock period = 4 system cycles), data_pattern[0] = 16'hAAAA (alternating 10), repetitions[0] = 1000.
- Start the generator via enable[0] = 1.
- cio_pcl0_tx_o produces a clock with period 4 cycles.
- cio_pda0_tx_o drives the data pattern, repeating 1000 times.
- When done, intr_done_ch0_o pulses; software reads status register to confirm completion and clears the interrupt.
